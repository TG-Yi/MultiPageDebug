## 📋 文档命名规范

为了保持文档的清晰性和可追溯性,我们采用以下命名规范:

### 功能文档
格式: `{功能名称}-v{版本号}.md`
- 示例: `auto-theme-v1.1.0.md`
- 说明: 清晰标识功能和引入版本

### 修复文档
格式: `{问题简述}-v{修复版本}-{日期}.md`
- 示例: `mobile-button-v1.3.1-v1.3.2-20250112.md`
- 说明: 包含问题、修复版本和日期

### 发布文档
格式: `v{版本号}-{日期}.md`
- 示例: `v1.3.2-20250112.md`
- 说明: 版本号和发布日期

### 测试文档
格式: `{测试内容}-v{版本号}.md`
- 示例: `mobile-drag-test-v1.3.2.md`
- 说明: 测试内容和相关版本

### 实现文档
格式: `{功能名称}-v{版本号}.md`
- 示例: `custom-schedule-v1.3.0.md`
- 说明: 功能名称和实现版本

## 🔍 如何查找文档

### 按功能查找
1. 查看 [features/](./features/) 目录
2. 根据功能名称和版本号定位文件

### 按版本查找
1. 查看 [releases/](./releases/) 目录
2. 找到对应版本的发布说明
3. 发布说明中包含该版本所有变更的链接

### 按问题查找
1. 查看 [fixes/](./fixes/) 目录
2. 根据问题描述或版本号查找

### 按时间查找
1. 文件名包含日期(格式: YYYYMMDD)
2. 按文件修改时间排序

## 📚 文档维护指南

### 添加新文档

1. **确定文档类型**,选择对应目录
2. **按照命名规范**创建文件
3. **更新本索引文件**,添加文档条目
4. **确保链接正确**,测试所有链接可访问

### 更新现有文档

1. **直接编辑**对应文档
2. **更新版本信息**(如适用)
3. **更新索引表格**(如有重大变更)
4. **提交时说明**变更内容

### 文档内容要求

每个文档应包含:
- 📌 **标题和版本**: 清晰的文档标题
- 📝 **概述**: 简要说明文档内容
- 🎯 **目标读者**: 适用人群
- 📖 **详细内容**: 结构化的正文
- 🔗 **相关链接**: 关联文档的链接
- 📅 **更新记录**: 文档修改历史
